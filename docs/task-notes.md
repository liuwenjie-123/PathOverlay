## T052 - 修复空嵌套目录创建提前落到真实 source

2026-04-30 复查测试机日志 `PathOverlay-Test-20260430-104715.log`：T043 兼容性场景中 `New-Item -ItemType Directory -Force source\empty-root\nested\leaf` 后，overlay 可见性和 shadow 目录创建均通过，但禁用 rule 后真实 source 下出现 `empty-root`，违反 copy-on-write 隔离。初步根因为驱动 `PathOverlayHasWriteIntent` 在 `Create.SecurityContext == NULL` 时直接返回 false，导致即使 create disposition 是 `FILE_OPEN_IF`/`FILE_CREATE`，也可能被当作普通目录打开；而不存在目录且 shadow 不存在时，普通目录打开分支会放行到真实 source。

2026-04-30：已调整驱动 `PathOverlayHasWriteIntent`，先根据 create disposition 判断 `FILE_CREATE`、`FILE_OPEN_IF`、`FILE_OVERWRITE`、`FILE_OVERWRITE_IF`、`FILE_SUPERSEDE`，再读取 `SecurityContext->DesiredAccess`。这样即使 `SecurityContext == NULL`，目录逐级创建也会按写入/创建意图走 shadow COW，不会因普通目录打开分支放行到真实 source。验证通过：`scripts/build.ps1`、`scripts/test.ps1`、`scripts/package-test-machine.ps1 -Configuration Release`；新测试机包已生成到 `test-machine-package`。当前本机启动项未显示 `testsigning Yes`，无法加载测试签名驱动跑真实 E2E，T052 暂 blocked，等待测试机复跑。
